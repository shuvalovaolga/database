Соглашения синтаксиса

Следующая таблица описывает соглашения, используемые при описании синтаксиса инструкций SQL и в коде примеров, и приводит примеры их использования:

Соглашение Назначение Пример
UPPERCASE
(ВЕРХНИЙ_РЕГИСТР)		 Ключевые слова, которые должны быть использованы точно так, как они встречаются в коде примеров. SET TERM !!;
italic
(наклонный_шрифт)		 Параметры, которые не могут быть разбиты на меньшие синтаксические единицы. Например, имя таблицы не может быть поделено на подэлементы. CREATE TABLE name (<col> [, <col> ...]);
<italic> Параметры в угловых скобках, которые могут быть разбиты на меньшие синтаксические единицы. Например, определение столбца (<col>) может включат в себя имя столбца, тип данных и определение ограничения. CREATE TABLE name (<col> [, <col> ...]);

<col> = name <datatype> [CONSTRAINT name <type>]
bold
(полужирный_шрифт)		 В описании синтаксиса, всегда часть синтаксической конструкции. Например, при описании размерности массива внешние скобки необходимы. <array_dim> = [x:y [, x:y ...]]
[ ] Квадратные скобки включают факультативный синтаксис. <col> [, <col> ...]
... Многоточие указывает, что предложение в скобках может быть повторено, по мере необходимости. (<col> [, <col> ...]);
| Символ канала обозначает, что любое из двух предложений, которые им разделены, может использоваться, но не оба одновременно.

Внутри фигурных скобок символ канала разделяет множественный выбор, один из которых должен быть использован.

SET TRANSACTION {SNAPSHOT [TABLE STABILITY] | READ COMMITTED};
{ } Фигурные скобки определяют, что одна из заключенных в них опций должна быть включена в фактическую инструкцию. SET TRANSACTION {SNAPSHOT [TABLE STABILITY] | READ COMMITTED};
