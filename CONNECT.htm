Описание

Инструкция CONNECT:

  • Инициализирует структуру данных базы данных.
  • Определяет находится ли база данных на узле возникновения (локальная база данных) или на другом узле (удаленная база данных). Базы данных используемые для клиентов Windows 3.1 всегда на удаленном сервере. Происходит сообщение об ошибках, если InterBase не может расположить базу данных.
  • Факультативно, определяет имя пользователя и пароль для использования когда присоединение к базе данных. Клиенты Windows 3.1 должны всегда отправлять допустимое имя пользователя и пароль.
  • Присоединяется к базе данных и проверяет страницу заголовка. Файл базы данных должен содержать допустимую базу данных, а так же номер версии дисковой структуру (ODS) базы данных должен быть распознан, установленной на сервере версиией InterBase, иначе InterBase возвращает ошибку.

Когда CONNECT присоединяет к базе данных, используется кодировка по умолчанию (NONE), иначе определенная в предшествующей инструкции SET NAMES. Каждый раз когда используется CONNECT для присоединения к базе данных, предыдущая присоединения разрываются.

К началу страницы...

Синтаксис

CONNECT ["]<filespec>["] [USER "username" [PASSWORD "password"]];

Обратите внимание:

Аргумент Описание
"<filespec>" Имя файла базы данных. Должно включать путь и узел.
USER "username" Строка, которая определяет имяя пользователя для присоединения к базе данных. Сервер проверяет имя пользователя в базе данных защиты isc.gdb. Имя пользователя не чувствительно к регистру.
PASSWORD "password" Строка, которая определяет пароль для присоединения к базе данных. Сервер проверяет имя пользователя и пароль в базе данных защиты isc.gdb. Чувствительно к регистру.

Примеры

Следующая инструкция открывает базу данных для использования. Она использует все доступные опции CONNECT:

CONNECT "employee.gdb" USER "ACCT_REC" PASSWORD "peanuts";
