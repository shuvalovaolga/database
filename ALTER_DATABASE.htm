ALTER DATABASE

Описание
Синтаксис
Примеры
Смотри так же

Описание

Инструкция ALTER DATABASE добавляет вторичные файлы к существующей базе данных. Вторичные файлы полезны для управления ростом и расположением базы данных. Они позволяют файлам базы данных быть распределенными на различных запоминающих устройствах, но при этом файлы должны оставаться на том же самом узле, что и первичный файл базы данных.

База данных может быть изменена ее создателем и пользователем SYSDBA.

ALERT DATABASE требует эксклюзивного доступа к базе данных.

На системе NetWare только, ALTER DATABASE также дает возможность вам изменить Write-ahead log (WAL) protocol.

Синтаксис

ALTER {DATABASE | SCHEMA}
ADD <add_clause>;

<add_clause> =
FILE "<filespec>" [<fileinfo>] [<add_clause>]

<fileinfo> = LENGTH [=] int [PAGE[S]] | STARTING [AT [PAGE]] int
Аргумент Описание
SCHEMA Альтернативное ключевое слово для DATABASE
ADD FILE "<filespec>" Добавляет один или более вторичных файлов, в которых будут размещены страницы базы данных, когда первичный файл будет заполнен.
LENGTH [=] int [PAGE[S]] Определяет диапазон страниц для вторичного файла, устанавливая количество страниц для каждого файла.
STARTING [AT [PAGE]] int Определяет диапазон страниц для вторичного файла, устанавливая стартовый номер страницы.

Примеры

Следующая инструкция добавляет вторичные файлы к существующей базе данных:

ALTER DATABASE
ADD FILE "employee.gd1"
  STARTING AT PAGE 10001
    LENGTH 10000
  ADD FILE "employee.gd2"
    LENGTH 10000;
